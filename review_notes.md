## Phase 1 - Data Analysis Foundations

### Week 1 Exercise 1 - Basic Dataset Inspection

완료 상태: 완료

사용한 데이터셋:

- `data/week1_macro_practice.csv`

실습 목표:

- 경제 데이터셋을 불러온다.
- 데이터의 행과 열 개수를 확인한다.
- 컬럼명과 자료형을 확인한다.
- 결측치와 중복 행을 확인한다.
- 날짜 컬럼과 한 행(row)의 의미를 설명한다.

배운 내용:

- 데이터 분석 전에 먼저 데이터 구조를 점검해야 한다.
- 한 행(row)이 무엇을 의미하는지 모르면 이후 분석과 모델링 해석이 흔들린다.
- 결측치(`NaN`)는 값이 비어 있는 상태를 의미하며, 모델링 전에 반드시 위치와 개수를 확인해야 한다.
- 중복 행은 특정 값 하나가 반복되는 것이 아니라, 한 행의 모든 컬럼 값이 다른 행의 모든 컬럼 값과 동일하게 반복되는 경우다.
- 코드 결과를 실행하는 것만으로는 충분하지 않고, 출력 결과를 문장으로 해석해야 한다.

사용한 함수/메서드:

#### `pd.read_csv()`

- 역할: CSV 파일을 Pandas DataFrame으로 불러온다.
- 왜 필요한가: 분석하려는 데이터를 Python/Pandas 안에서 다룰 수 있는 표 형태로 가져오기 위해 필요하다.
- 주요 인수:
  - 첫 번째 인수 또는 `filepath_or_buffer`: 불러올 CSV 파일 경로.
  - 예: `"data/week1_macro_practice.csv"`
- 반환값: CSV 내용을 담은 `DataFrame`.
- 원본을 바꾸는가: 외부 CSV 파일 자체를 바꾸지 않는다.
- 다시 저장해야 하는가: 반환된 DataFrame을 변수에 저장해야 이후 분석에 사용할 수 있다.
- 예시 형태:
  - `data_set = pd.read_csv("data/week1_macro_practice.csv")`

#### `.head()`

- 역할: DataFrame의 앞부분 일부 행을 보여준다.
- 왜 필요한가: 전체 데이터를 보기 전에 데이터가 어떤 모양인지 빠르게 확인하기 위해 필요하다.
- 주요 인수:
  - `n`: 몇 개 행을 볼지 지정한다. 기본값은 `5`.
  - 예: `.head()`는 앞 5행, `.head(10)`은 앞 10행을 보여준다.
- 반환값: 앞부분 일부 행으로 구성된 새 DataFrame.
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이므로 다시 저장하지 않아도 된다.

#### `.shape`

- 역할: DataFrame의 행 개수와 열 개수를 알려준다.
- 왜 필요한가: 데이터 크기를 먼저 파악해야 분석 규모와 구조를 이해할 수 있다.
- 주요 인수: 없다. 메서드가 아니라 속성(attribute)이므로 괄호를 붙이지 않는다.
- 반환값: `(행 개수, 열 개수)` 형태의 튜플(tuple).
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이므로 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `(49, 8)`은 49개 행과 8개 열이 있다는 뜻이다.

#### `.columns`

- 역할: DataFrame의 컬럼명 목록을 보여준다.
- 왜 필요한가: 어떤 변수들이 데이터에 들어 있는지 확인하기 위해 필요하다.
- 주요 인수: 없다. 속성이므로 괄호를 붙이지 않는다.
- 반환값: 컬럼명들이 담긴 `Index` 객체.
- 원본을 바꾸는가: 확인만 하면 원본을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이므로 저장하지 않아도 된다.

#### `.dtypes`

- 역할: 각 컬럼의 자료형을 보여준다.
- 왜 필요한가: 숫자인지, 문자열인지, 날짜인지에 따라 가능한 분석 방법이 달라지기 때문에 필요하다.
- 주요 인수: 없다. 속성이므로 괄호를 붙이지 않는다.
- 반환값: 컬럼별 자료형이 담긴 `Series`.
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이므로 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `date`는 처음에 `str`이었다.
  - 나머지 경제 지표 컬럼들은 `float64`였다.

#### `.isna()`

- 역할: 각 값이 결측치인지 확인한다.
- 왜 필요한가: 결측치가 있는 위치를 확인하기 위해 필요하다.
- 주요 인수: 보통 사용하지 않는다.
- 반환값: 원래 DataFrame과 같은 모양의 Boolean DataFrame.
  - 결측치이면 `True`
  - 결측치가 아니면 `False`
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 `.sum()`과 함께 확인하므로 따로 저장하지 않아도 된다.

#### `.sum()`

- 역할: 값을 더한다.
- 왜 필요한가: `True`/`False` 결과에서 `True` 개수를 세기 위해 자주 사용한다.
- 주요 인수:
  - 기본적으로 열 방향으로 합계를 계산한다.
  - 지금 단계에서는 인수를 몰라도 된다.
- 반환값: 합계 결과.
- 원본을 바꾸는가: 원본을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이면 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `data_set.isna().sum()`은 컬럼별 결측치 개수를 센다.
  - Pandas에서 `True`는 계산할 때 1, `False`는 0처럼 취급되기 때문에 가능하다.

#### `.duplicated()`

- 역할: 각 행이 이전 행과 중복되는지 확인한다.
- 왜 필요한가: 같은 관측치가 반복되어 들어간 문제를 찾기 위해 필요하다.
- 주요 인수:
  - 기본 사용에서는 인수를 넣지 않아도 된다.
  - 지금 단계에서는 전체 행 기준 중복 여부를 확인한다고 이해하면 된다.
- 반환값: Boolean Series.
  - 중복 행이면 `True`
  - 중복 행이 아니면 `False`
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 `.sum()`과 함께 확인하므로 따로 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `data_set.duplicated().sum()`은 중복 행의 개수를 센다.

수정된 실수:

- 처음에는 특정 값이 반복되는 것과 중복 행을 혼동했다.
- 이후 중복 행은 "한 행 전체가 다른 행 전체와 동일한 경우"라고 정리했다.

핵심 정리:

- 데이터 분석의 첫 단계는 모델링이 아니라 데이터 구조 점검이다.
- 출력 결과를 숫자로만 보는 것이 아니라, 문장으로 설명할 수 있어야 한다.

### Week 1 Exercise 2 - Date Conversion and Chronological Sorting

완료 상태: 완료

사용한 데이터셋:

- `data/week1_macro_practice.csv`

실습 목표:

- `date` 컬럼이 문자열인지 확인한다.
- `date` 컬럼을 Pandas가 이해하는 날짜 타입으로 변환한다.
- 날짜 기준으로 오름차순 정렬한다.
- 날짜가 실제로 시간순으로 정렬되었는지 확인한다.
- 가장 이른 날짜와 가장 늦은 날짜를 해석한다.

배운 내용:

- 사람 눈에는 날짜처럼 보여도 Pandas에서는 문자열일 수 있다.
- 날짜 컬럼이 문자열이면 시간 연산, 기간 필터링, 시계열 정렬, 시계열 그래프, 예측 평가에서 문제가 생길 수 있다.
- 날짜 타입 변환과 날짜순 정렬은 서로 다른 작업이다.
- `sort_values()`는 정렬된 새 결과를 반환하므로, 계속 사용하려면 변수에 다시 저장해야 한다.
- 가장 빠른 날짜는 가장 오래된 날짜이고, 가장 늦은 날짜는 가장 최근 날짜다.
- 2021년 1월부터 2024년 12월까지는 약 4년의 월별 데이터다.

사용한 함수/메서드:

#### `pd.to_datetime()`

- 역할: 문자열이나 다른 형식으로 저장된 날짜 값을 Pandas의 날짜 타입으로 변환한다.
- 왜 필요한가: Pandas가 해당 컬럼을 진짜 시간 정보로 인식해야 시간순 정렬, 기간 필터링, 월/연도 추출, 시계열 분석을 할 수 있기 때문이다.
- 주요 인수:
  - 첫 번째 인수: 날짜로 변환할 값 또는 컬럼.
  - 예: `data_set.date`
- 반환값: 날짜 타입으로 변환된 Series 또는 DatetimeIndex.
- 원본을 바꾸는가: 원본 컬럼을 자동으로 바꾸지 않는다.
- 다시 저장해야 하는가: 변환 결과를 계속 사용하려면 원래 컬럼에 다시 저장해야 한다.
- 예시 형태:
  - `data_set.date = pd.to_datetime(data_set.date)`
- 이번 실습에서의 의미:
  - `date` 컬럼이 `str`에서 `datetime64[us]`로 바뀌었다.

#### `sort_values()`

- 역할: DataFrame을 특정 컬럼 기준으로 정렬한다.
- 왜 필요한가: 시계열 데이터는 시간 순서가 중요하므로 날짜 기준으로 과거에서 최근 순서로 정렬해야 한다.
- 주요 인수:
  - `by`: 어떤 컬럼을 기준으로 정렬할지 지정한다.
    - 예: `by="date"`
  - `ascending`: 오름차순 여부를 지정한다.
    - `True`: 작은 값에서 큰 값, 날짜라면 과거에서 최근 순서.
    - `False`: 큰 값에서 작은 값, 날짜라면 최근에서 과거 순서.
- 반환값: 정렬된 새 DataFrame.
- 원본을 바꾸는가: 기본적으로 원본 DataFrame을 직접 바꾸지 않는다.
- 다시 저장해야 하는가: 정렬 결과를 계속 사용하려면 변수에 다시 저장해야 한다.
- 예시 형태:
  - `data_set = data_set.sort_values(by="date", ascending=True)`
- 이번 실습에서의 의미:
  - `date` 기준으로 2021-01-31부터 2024-12-31까지 오름차순 정렬했다.

#### `.is_monotonic_increasing`

- 역할: 값이 앞에서 뒤로 갈수록 같거나 증가하는 순서인지 확인한다.
- 왜 필요한가: 날짜 컬럼이 실제로 오름차순으로 정렬되어 있는지 눈으로 보지 않고 확인하기 위해 필요하다.
- 주요 인수: 없다. 속성이므로 괄호를 붙이지 않는다.
- 반환값: Boolean 값.
  - 정렬되어 있으면 `True`
  - 정렬되어 있지 않으면 `False`
- 원본을 바꾸는가: 원본을 바꾸지 않는다.
- 다시 저장해야 하는가: 확인용이므로 보통 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `data_set.date.is_monotonic_increasing` 결과가 `True`라면 날짜가 오름차순이라는 뜻이다.

수정된 실수:

- 처음에는 가장 빠른 날짜와 가장 늦은 날짜를 반대로 해석했다.
- 처음에는 `sort_values()`가 반환하는 정렬 결과를 다시 변수에 저장해야 하는지 헷갈렸다.
- 이후 정렬 결과를 계속 쓰려면 `data_set = data_set.sort_values(...)`처럼 다시 저장해야 한다고 정리했다.

핵심 정리:

- 날짜 변환은 "문자열을 날짜로 인식하게 만드는 작업"이다.
- 날짜 정렬은 "시간 순서를 맞추는 작업"이다.
- 시계열 데이터에서는 날짜 타입과 정렬 상태가 이후 분석과 예측 평가의 기초가 된다.
