## Phase 1 - Data Analysis Foundations

### Week 1 Exercise 1 - Basic Dataset Inspection

완료 상태: 완료

사용한 데이터셋:

- `data/week1_macro_practice.csv`

실습 목표:

- 경제 데이터셋을 불러온다.
- 데이터의 행과 열 개수를 확인한다.
- 컬럼명과 자료형을 확인한다.
- 결측치와 중복 행을 확인한다.
- 날짜 컬럼과 한 행(row)의 의미를 설명한다.

배운 내용:

- 데이터 분석 전에 먼저 데이터 구조를 점검해야 한다.
- 한 행(row)이 무엇을 의미하는지 모르면 이후 분석과 모델링 해석이 흔들린다.
- 결측치(`NaN`)는 값이 비어 있는 상태를 의미하며, 모델링 전에 반드시 위치와 개수를 확인해야 한다.
- 중복 행은 특정 값 하나가 반복되는 것이 아니라, 한 행의 모든 컬럼 값이 다른 행의 모든 컬럼 값과 동일하게 반복되는 경우다.
- 코드 결과를 실행하는 것만으로는 충분하지 않고, 출력 결과를 문장으로 해석해야 한다.

사용한 함수/메서드:

#### `pd.read_csv()`

- 역할: CSV 파일을 Pandas DataFrame으로 불러온다.
- 왜 필요한가: 분석하려는 데이터를 Python/Pandas 안에서 다룰 수 있는 표 형태로 가져오기 위해 필요하다.
- 주요 인수:
  - 첫 번째 인수 또는 `filepath_or_buffer`: 불러올 CSV 파일 경로.
  - 예: `"data/week1_macro_practice.csv"`
- 반환값: CSV 내용을 담은 `DataFrame`.
- 원본을 바꾸는가: 외부 CSV 파일 자체를 바꾸지 않는다.
- 다시 저장해야 하는가: 반환된 DataFrame을 변수에 저장해야 이후 분석에 사용할 수 있다.
- 예시 형태:
  - `data_set = pd.read_csv("data/week1_macro_practice.csv")`

#### `.head()`

- 역할: DataFrame의 앞부분 일부 행을 보여준다.
- 왜 필요한가: 전체 데이터를 보기 전에 데이터가 어떤 모양인지 빠르게 확인하기 위해 필요하다.
- 주요 인수:
  - `n`: 몇 개 행을 볼지 지정한다. 기본값은 `5`.
  - 예: `.head()`는 앞 5행, `.head(10)`은 앞 10행을 보여준다.
- 반환값: 앞부분 일부 행으로 구성된 새 DataFrame.
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이므로 다시 저장하지 않아도 된다.

#### `.shape`

- 역할: DataFrame의 행 개수와 열 개수를 알려준다.
- 왜 필요한가: 데이터 크기를 먼저 파악해야 분석 규모와 구조를 이해할 수 있다.
- 주요 인수: 없다. 메서드가 아니라 속성(attribute)이므로 괄호를 붙이지 않는다.
- 반환값: `(행 개수, 열 개수)` 형태의 튜플(tuple).
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이므로 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `(49, 8)`은 49개 행과 8개 열이 있다는 뜻이다.

#### `.columns`

- 역할: DataFrame의 컬럼명 목록을 보여준다.
- 왜 필요한가: 어떤 변수들이 데이터에 들어 있는지 확인하기 위해 필요하다.
- 주요 인수: 없다. 속성이므로 괄호를 붙이지 않는다.
- 반환값: 컬럼명들이 담긴 `Index` 객체.
- 원본을 바꾸는가: 확인만 하면 원본을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이므로 저장하지 않아도 된다.

#### `.dtypes`

- 역할: 각 컬럼의 자료형을 보여준다.
- 왜 필요한가: 숫자인지, 문자열인지, 날짜인지에 따라 가능한 분석 방법이 달라지기 때문에 필요하다.
- 주요 인수: 없다. 속성이므로 괄호를 붙이지 않는다.
- 반환값: 컬럼별 자료형이 담긴 `Series`.
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이므로 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `date`는 처음에 `str`이었다.
  - 나머지 경제 지표 컬럼들은 `float64`였다.

#### `.isna()`

- 역할: 각 값이 결측치인지 확인한다.
- 왜 필요한가: 결측치가 있는 위치를 확인하기 위해 필요하다.
- 주요 인수: 보통 사용하지 않는다.
- 반환값: 원래 DataFrame과 같은 모양의 Boolean DataFrame.
  - 결측치이면 `True`
  - 결측치가 아니면 `False`
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 `.sum()`과 함께 확인하므로 따로 저장하지 않아도 된다.

#### `.sum()`

- 역할: 값을 더한다.
- 왜 필요한가: `True`/`False` 결과에서 `True` 개수를 세기 위해 자주 사용한다.
- 주요 인수:
  - 기본적으로 열 방향으로 합계를 계산한다.
  - 지금 단계에서는 인수를 몰라도 된다.
- 반환값: 합계 결과.
- 원본을 바꾸는가: 원본을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 확인용이면 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `data_set.isna().sum()`은 컬럼별 결측치 개수를 센다.
  - Pandas에서 `True`는 계산할 때 1, `False`는 0처럼 취급되기 때문에 가능하다.

#### `.duplicated()`

- 역할: 각 행이 이전 행과 중복되는지 확인한다.
- 왜 필요한가: 같은 관측치가 반복되어 들어간 문제를 찾기 위해 필요하다.
- 주요 인수:
  - 기본 사용에서는 인수를 넣지 않아도 된다.
  - 지금 단계에서는 전체 행 기준 중복 여부를 확인한다고 이해하면 된다.
- 반환값: Boolean Series.
  - 중복 행이면 `True`
  - 중복 행이 아니면 `False`
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 보통 `.sum()`과 함께 확인하므로 따로 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `data_set.duplicated().sum()`은 중복 행의 개수를 센다.

수정된 실수:

- 처음에는 특정 값이 반복되는 것과 중복 행을 혼동했다.
- 이후 중복 행은 "한 행 전체가 다른 행 전체와 동일한 경우"라고 정리했다.

핵심 정리:

- 데이터 분석의 첫 단계는 모델링이 아니라 데이터 구조 점검이다.
- 출력 결과를 숫자로만 보는 것이 아니라, 문장으로 설명할 수 있어야 한다.

### Week 1 Exercise 2 - Date Conversion and Chronological Sorting

완료 상태: 완료

사용한 데이터셋:

- `data/week1_macro_practice.csv`

실습 목표:

- `date` 컬럼이 문자열인지 확인한다.
- `date` 컬럼을 Pandas가 이해하는 날짜 타입으로 변환한다.
- 날짜 기준으로 오름차순 정렬한다.
- 날짜가 실제로 시간순으로 정렬되었는지 확인한다.
- 가장 이른 날짜와 가장 늦은 날짜를 해석한다.

배운 내용:

- 사람 눈에는 날짜처럼 보여도 Pandas에서는 문자열일 수 있다.
- 날짜 컬럼이 문자열이면 시간 연산, 기간 필터링, 시계열 정렬, 시계열 그래프, 예측 평가에서 문제가 생길 수 있다.
- 날짜 타입 변환과 날짜순 정렬은 서로 다른 작업이다.
- `sort_values()`는 정렬된 새 결과를 반환하므로, 계속 사용하려면 변수에 다시 저장해야 한다.
- 가장 빠른 날짜는 가장 오래된 날짜이고, 가장 늦은 날짜는 가장 최근 날짜다.
- 2021년 1월부터 2024년 12월까지는 약 4년의 월별 데이터다.

사용한 함수/메서드:

#### `pd.to_datetime()`

- 역할: 문자열이나 다른 형식으로 저장된 날짜 값을 Pandas의 날짜 타입으로 변환한다.
- 왜 필요한가: Pandas가 해당 컬럼을 진짜 시간 정보로 인식해야 시간순 정렬, 기간 필터링, 월/연도 추출, 시계열 분석을 할 수 있기 때문이다.
- 주요 인수:
  - 첫 번째 인수: 날짜로 변환할 값 또는 컬럼.
  - 예: `data_set.date`
- 반환값: 날짜 타입으로 변환된 Series 또는 DatetimeIndex.
- 원본을 바꾸는가: 원본 컬럼을 자동으로 바꾸지 않는다.
- 다시 저장해야 하는가: 변환 결과를 계속 사용하려면 원래 컬럼에 다시 저장해야 한다.
- 예시 형태:
  - `data_set.date = pd.to_datetime(data_set.date)`
- 이번 실습에서의 의미:
  - `date` 컬럼이 `str`에서 `datetime64[us]`로 바뀌었다.

#### `sort_values()`

- 역할: DataFrame을 특정 컬럼 기준으로 정렬한다.
- 왜 필요한가: 시계열 데이터는 시간 순서가 중요하므로 날짜 기준으로 과거에서 최근 순서로 정렬해야 한다.
- 주요 인수:
  - `by`: 어떤 컬럼을 기준으로 정렬할지 지정한다.
    - 예: `by="date"`
  - `ascending`: 오름차순 여부를 지정한다.
    - `True`: 작은 값에서 큰 값, 날짜라면 과거에서 최근 순서.
    - `False`: 큰 값에서 작은 값, 날짜라면 최근에서 과거 순서.
- 반환값: 정렬된 새 DataFrame.
- 원본을 바꾸는가: 기본적으로 원본 DataFrame을 직접 바꾸지 않는다.
- 다시 저장해야 하는가: 정렬 결과를 계속 사용하려면 변수에 다시 저장해야 한다.
- 예시 형태:
  - `data_set = data_set.sort_values(by="date", ascending=True)`
- 이번 실습에서의 의미:
  - `date` 기준으로 2021-01-31부터 2024-12-31까지 오름차순 정렬했다.

#### `.is_monotonic_increasing`

- 역할: 값이 앞에서 뒤로 갈수록 같거나 증가하는 순서인지 확인한다.
- 왜 필요한가: 날짜 컬럼이 실제로 오름차순으로 정렬되어 있는지 눈으로 보지 않고 확인하기 위해 필요하다.
- 주요 인수: 없다. 속성이므로 괄호를 붙이지 않는다.
- 반환값: Boolean 값.
  - 정렬되어 있으면 `True`
  - 정렬되어 있지 않으면 `False`
- 원본을 바꾸는가: 원본을 바꾸지 않는다.
- 다시 저장해야 하는가: 확인용이므로 보통 저장하지 않아도 된다.
- 이번 실습에서의 의미:
  - `data_set.date.is_monotonic_increasing` 결과가 `True`라면 날짜가 오름차순이라는 뜻이다.

수정된 실수:

- 처음에는 가장 빠른 날짜와 가장 늦은 날짜를 반대로 해석했다.
- 처음에는 `sort_values()`가 반환하는 정렬 결과를 다시 변수에 저장해야 하는지 헷갈렸다.
- 이후 정렬 결과를 계속 쓰려면 `data_set = data_set.sort_values(...)`처럼 다시 저장해야 한다고 정리했다.

핵심 정리:

- 날짜 변환은 "문자열을 날짜로 인식하게 만드는 작업"이다.
- 날짜 정렬은 "시간 순서를 맞추는 작업"이다.
- 시계열 데이터에서는 날짜 타입과 정렬 상태가 이후 분석과 예측 평가의 기초가 된다.

### Week 1 Exercise 3 - Missing Value Analysis

완료 상태: 완료

사용한 데이터셋:

- `data/week1_macro_practice.csv`

실습 목표:

- 컬럼별 결측치 개수를 확인한다.
- 결측치가 있는 실제 행을 확인한다.
- 어떤 날짜의 어떤 경제 지표가 비어 있는지 설명한다.
- 결측치를 바로 삭제하거나 채우지 않는 이유를 설명한다.
- 시계열 데이터에서 결측치 처리와 데이터 누수(Data Leakage)의 관계를 인식한다.

배운 내용:

- 결측치는 단순한 빈칸이 아니라 분석 목적에 따라 처리 방식이 달라지는 데이터 품질 문제다.
- 결측치가 있는 컬럼은 `industrial_production_index`, `pmi`, `imports_usd_billion`이었다.
- 각 결측치는 1개씩 있었다.
- 결측치가 있는 날짜는 `2023-05-31`, `2023-09-30`, `2024-09-30`이었다.
- 산업생산지수는 분석 중심 지표 후보이므로 결측치가 있으면 추세 분석이나 예측 문제 정의에 영향을 줄 수 있다.
- PMI와 수입액은 관련 경제 지표 후보이므로 나중에 설명 변수로 사용할 경우 결측 처리 방식이 필요하다.
- 결측치를 바로 삭제하면 특정 월의 관측치를 잃을 수 있다.
- 평균값으로 바로 채우면 실제 시계열 흐름을 왜곡할 수 있다.
- 시계열 데이터에서 미래 값을 이용해 과거 결측치를 채우면 데이터 누수(Data Leakage)가 생길 수 있다.

사용한 함수/메서드:

#### `.loc`

- 역할: DataFrame에서 원하는 행과 열을 조건으로 선택한다.
- 왜 필요한가: 전체 데이터 중에서 특정 조건에 맞는 행만 확인하거나, 특정 행/열 조합을 선택하기 위해 필요하다.
- 기본 형태:
  - `data_set.loc[행 조건]`
  - `data_set.loc[행 조건, 열 선택]`
- 주요 인수:
  - 첫 번째 위치: 선택할 행 조건.
    - 예: `data_set["pmi"].isna()`는 `pmi`가 결측치인 행만 선택하는 조건이다.
  - 두 번째 위치: 선택할 열.
    - 생략하면 모든 열을 보여준다.
    - 예: `["date", "pmi"]`처럼 특정 열만 선택할 수도 있다.
- 반환값: 조건에 맞는 행과 열로 구성된 Series 또는 DataFrame.
- 원본을 바꾸는가: 단순 조회만 하면 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 확인용이면 저장하지 않아도 된다. 선택한 결과를 나중에 다시 쓰려면 변수에 저장할 수 있다.
- 이번 실습에서의 의미:
  - 결측치가 있는 실제 행을 확인하기 위해 사용했다.
  - 예: `data_set.loc[data_set.isna().any(axis=1)]`는 결측치가 하나라도 있는 행을 보여준다.

#### `.any()`

- 역할: 여러 Boolean 값 중 하나라도 `True`가 있는지 확인한다.
- 왜 필요한가: 한 행 안에 여러 컬럼 중 결측치가 하나라도 있는지 확인하기 위해 필요하다.
- 주요 인수:
  - `axis=0`: 열 방향으로 검사한다. 기본값이다. 각 컬럼에 `True`가 하나라도 있는지 확인한다.
  - `axis=1`: 행 방향으로 검사한다. 각 행에 `True`가 하나라도 있는지 확인한다.
- 반환값: Boolean Series.
  - 조건을 만족하면 `True`
  - 만족하지 않으면 `False`
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 확인용이면 저장하지 않아도 된다. 조건을 재사용하려면 변수에 저장할 수 있다.
- 이번 실습에서의 의미:
  - `data_set.isna()`는 각 칸이 결측치인지 `True`/`False`로 표시한다.
  - `data_set.isna().any(axis=1)`는 각 행을 가로로 보면서 결측치가 하나라도 있으면 `True`를 반환한다.
  - 이 결과를 `.loc[]` 안에 넣으면 결측치가 있는 행만 볼 수 있다.

#### `data_set.isna().any(axis=1)`

- 역할: 각 행에 결측치가 하나라도 있는지 확인하는 조건을 만든다.
- 왜 필요한가: 결측치가 있는 전체 행을 한 번에 확인하기 위해 필요하다.
- 구성:
  - `data_set.isna()`: 각 칸이 결측치인지 확인한다.
  - `.any(axis=1)`: 각 행에서 결측치가 하나라도 있는지 확인한다.
- 반환값: 행마다 `True` 또는 `False`가 담긴 Boolean Series.
- 원본을 바꾸는가: 원본을 바꾸지 않는다.
- 다시 저장해야 하는가: `.loc[]` 안에서 바로 사용할 수 있고, 재사용하려면 변수에 저장할 수 있다.
- 이번 실습에서의 의미:
  - 결측치가 있는 3개 행을 한 번에 찾는 조건으로 사용했다.

수정된 실수:

- 처음에는 `policy_rate`도 결측치가 있는 경제 지표라고 적었지만, 실제로는 결측치가 없었다.
- 처음에는 결측치를 나중에 한 번에 처리하기 위해 지금 처리하지 않는다고 설명했지만, 이후 결측치 처리 방식은 컬럼의 의미와 분석 목적에 따라 달라진다고 수정했다.

핵심 정리:

- 결측치 분석은 결측치 개수를 세는 것에서 끝나지 않는다.
- 어떤 날짜의 어떤 지표가 비어 있는지 확인해야 한다.
- 결측치 처리는 기계적으로 삭제하거나 평균으로 채우는 작업이 아니다.
- 시계열 데이터에서는 결측치를 채우는 방식이 데이터 누수와 연결될 수 있다.

### Week 1 Exercise 5 - Basic Time-Series Visualization

실습 목표:

- 결측치가 선 그래프에 어떤 영향을 줄 수 있는지 설명한다.
- 통계 요약표와 그래프가 각각 어떤 정보를 더 잘 보여주는지 구분한다.
- 그래프만 보고 알 수 있는 정보와 원본 데이터를 확인해야 알 수 있는 정보를 구분한다.

배운 내용:

- 선 그래프는 상승, 하락, 전환점, 끊김 같은 흐름을 요약표보다 직관적으로 보여준다.
- 요약 통계는 평균, 최솟값, 최댓값 같은 정확한 수치를 확인하는 데 유용하지만, 시간 순서의 흐름은 잘 보여주지 못한다.
- 결측치가 있으면 선 그래프에서 특정 구간이 끊겨 보일 수 있고, 흐름을 해석할 때 주의해야 한다.
- 그래프 축에 모든 날짜가 표시되지 않으면 정확한 결측 월을 그래프만으로 단정하면 안 된다.
- 정확한 결측 날짜나 특정 값의 날짜는 원본 DataFrame을 다시 확인해야 한다.

사용한 함수/메서드:

#### `.plot()`

- 역할: DataFrame이나 Series의 데이터를 그래프로 그린다.
- 왜 필요한가: 숫자 요약만으로는 시간 흐름, 상승/하락 패턴, 끊김, 급격한 변화 등을 파악하기 어렵기 때문에 시각화가 필요하다.
- 기본 형태:
  - `data_set.plot(x="date", y="pmi")`
- 주요 인수:
  - `x`: x축에 사용할 컬럼 이름을 지정한다.
    - 예: `x="date"`
    - 여기서 `"date"`는 아무 문자열이 아니라 DataFrame 안에 있는 컬럼 이름이다.
  - `y`: y축에 사용할 컬럼 이름을 지정한다.
    - 예: `y="pmi"` 또는 `y="industrial_production_index"`
  - `kind`: 어떤 종류의 그래프를 그릴지 지정한다.
    - 예: `kind="line"`은 선 그래프를 의미한다.
    - 시계열 흐름을 볼 때는 선 그래프가 자주 사용된다.
  - `title`: 그래프 제목을 지정한다.
    - 예: `title="PMI"`
- 반환값: 그래프 축 객체인 `Axes` 객체를 반환한다.
  - 지금 단계에서는 반환값을 깊게 다룰 필요는 없다.
  - 나중에 축 이름, 범례, 크기 등을 세밀하게 수정할 때 이 객체를 사용할 수 있다.
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가:
  - 단순히 그래프를 확인하는 목적이면 저장하지 않아도 된다.
  - 그래프를 추가로 수정하거나 재사용하려면 변수에 저장할 수 있다.
  - 예: `ax = data_set.plot(x="date", y="pmi")`
- 이번 실습에서의 의미:
  - `data_set.plot(x="date", y="industrial_production_index", kind="line")`은 날짜를 x축, 산업생산지수를 y축으로 하는 선 그래프를 만든다.
  - `data_set.plot(x="date", y="pmi", kind="line")`은 날짜를 x축, PMI를 y축으로 하는 선 그래프를 만든다.

#### `x="date"`와 `y="pmi"`에서 문자열을 쓰는 이유

- 역할: `plot()`에게 어떤 컬럼을 축으로 사용할지 컬럼 이름으로 알려준다.
- 왜 필요한가: `plot()`은 이미 `data_set.plot(...)`처럼 어떤 DataFrame에서 작업하는지 알고 있다. 따라서 실제 컬럼 데이터를 직접 넘기기보다, 그 DataFrame 안의 컬럼 이름을 문자열로 지정하는 방식이 자연스럽다.
- 중요한 구분:
  - `data_set["date"]`: `date` 컬럼의 실제 데이터를 꺼낸다.
  - `data_set.date`: `date` 컬럼의 실제 데이터를 꺼내는 편의 문법이다. 컬럼명이 단순할 때만 안전하다.
  - `x="date"`: `plot()`에게 x축으로 사용할 컬럼 이름이 `date`라고 알려준다.
- 왜 `data_set["date"]`가 더 안전한가:
  - 컬럼 이름에 공백이 있거나 Python 속성 이름으로 쓸 수 없는 문자가 있으면 `data_set.date` 방식은 사용할 수 없다.
  - 컬럼명이 DataFrame의 기존 메서드 이름과 겹치면 혼동이 생길 수 있다.
  - 그래서 컬럼을 직접 꺼낼 때는 `data_set["date"]`가 가장 명확하고 안전하다.
- 이번 실습에서의 의미:
  - `plot(x="date", y="pmi")`는 `date`와 `pmi`라는 실제 값을 쓰는 것이 아니라, `plot()`이 DataFrame 내부에서 해당 컬럼을 찾아 쓰도록 컬럼 이름을 넘기는 것이다.

### Week 1 Exercise 6 - Scatter Plot Relationship Check

실습 목표:

배운 내용:

- 산점도의 점 하나는 한 관측치, 이번 데이터에서는 한 달의 `pmi`와 `industrial_production_index` 조합을 의미한다.
- 점들이 오른쪽 위로 모이면 양의 관계가 있어 보일 수 있고, 오른쪽 아래로 모이면 음의 관계가 있어 보일 수 있다.
- 점들이 뚜렷한 방향 없이 흩어져 있으면 이 그래프만으로는 강한 관계가 보인다고 말하기 어렵다.

- 시계열 모델링에서는 같은 달의 관계보다 이전 달 값이 이후 값을 설명하는 시차 관계가 중요할 수 있다.
- 결측치가 있으면 해당 행은 산점도에서 점으로 표시되지 않을 수 있다.
- 산점도만으로 인과관계는 말할 수 없다.

사용한 함수/메서드:

#### `.plot(kind="scatter")`

- 역할: 두 숫자 변수의 관계를 산점도로 그린다.
- 왜 필요한가: 모델링 전에 한 변수가 커질 때 다른 변수도 함께 커지는지, 반대로 움직이는지, 아니면 뚜렷한 관계가 없는지 눈으로 확인하기 위해 필요하다.
- 기본 형태:
  - `data_set.plot(x="pmi", y="industrial_production_index", kind="scatter")`
- 주요 인수:
  - `x`: x축에 사용할 컬럼 이름을 지정한다.
    - 이번 실습에서는 `x="pmi"`를 사용했다.
  - `y`: y축에 사용할 컬럼 이름을 지정한다.
    - 이번 실습에서는 `y="industrial_production_index"`를 사용했다.
  - `kind`: 그래프 종류를 지정한다.
    - `kind="scatter"`는 산점도를 의미한다.
- 반환값: 그래프 축 객체인 `Axes` 객체를 반환한다.
  - 지금 단계에서는 반환값을 저장하지 않아도 된다.
  - 나중에 제목, 축 이름, 색상 등을 세밀하게 조정하려면 변수에 저장해 사용할 수 있다.
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가:
  - 단순 확인용이면 저장하지 않아도 된다.
  - 그래프를 추가로 수정하려면 `ax = data_set.plot(...)`처럼 저장할 수 있다.
- 이번 실습에서의 의미:
  - `pmi`가 높거나 낮을 때 `industrial_production_index`가 어떤 수준에 있는지 점들의 분포로 확인했다.

#### 산점도에서 `x`와 `y`의 의미

- 역할: 두 변수 중 어떤 변수를 가로축에 놓고, 어떤 변수를 세로축에 놓을지 정한다.
- 왜 필요한가: 산점도는 두 숫자 변수의 쌍을 점으로 나타내므로, x축과 y축을 명확히 정해야 해석할 수 있다.
- 주요 인수:
  - `x="pmi"`: 각 점의 가로 위치는 PMI 값으로 결정된다.
  - `y="industrial_production_index"`: 각 점의 세로 위치는 산업생산지수 값으로 결정된다.
- 반환값: 이 인수들 자체가 값을 반환하지는 않는다. `plot()`이 이 정보를 사용해서 그래프를 만든다.
- 원본을 바꾸는가: 원본 DataFrame을 바꾸지 않는다.
- 다시 저장해야 하는가: 축 지정 인수이므로 따로 저장하지 않는다.

수정된 실수:

- 처음에는 산점도에서 뚜렷한 관계가 보이지 않는다는 이유로 PMI가 예측에 도움이 되지 않는다고 강하게 표현했다.
- 이후 "이 산점도만으로는 PMI가 예측에 도움이 된다고 판단하기 어렵다"로 표현을 조정했다.
- 산점도에서 결측치가 있으면 선이 끊기는 것이 아니라, 해당 관측치의 점이 표시되지 않을 수 있다고 정리했다.

핵심 정리:

### Week 1 Review Check - End-to-End EDA Review

완료 상태: 완료

사용한 데이터셋:

- `data/week1_macro_practice.csv`

실습 목표:

- Week 1에서 배운 EDA 흐름을 처음부터 끝까지 다시 수행한다.
- 데이터 구조 확인, 날짜 처리, 결측치 확인, 요약 통계, 시각화, 모델링 전 판단을 하나의 절차로 연결한다.
- Week 2 모델링 기초로 넘어가기 전에 필수 개념이 연결되어 있는지 확인한다.

배운 내용:

- EDA는 개별 함수 사용이 아니라, 데이터를 모델링 가능한 상태로 이해하는 과정이다.
- 날짜 변환과 날짜 정렬은 다르다.
  - 날짜 변환은 문자열을 Pandas가 시간 정보로 인식하게 만드는 작업이다.
  - 날짜 정렬은 관측치를 과거에서 최근 순서로 맞추는 작업이다.
- 결측치는 개수만 세는 것이 아니라, 어떤 행과 어떤 컬럼에서 생겼는지 확인해야 한다.
- `describe()`의 `count`는 전체 행 수가 아니라 `NaN`을 제외한 값의 개수다.
- 평균은 전체 값을 하나의 숫자로 요약하므로 시간 순서를 보여주지 못한다.
- 선 그래프는 시간 흐름을 보기 좋고, 산점도는 두 변수 사이의 관계를 보기 좋다.
- 그래프만으로 원인, 인과관계, 예측력을 확정하면 안 된다.
- 예측 대상과 설명 변수는 데이터가 깨끗한지만 보고 고르는 것이 아니다.

사용한 함수/메서드:

#### Week 1 핵심 함수/메서드 한 줄 복습

- `pd.read_csv()`: CSV 파일을 Pandas DataFrame으로 불러온다.
- `.head()`: 데이터 앞부분을 확인한다.
- `.shape`: 행과 열 개수를 확인한다.
- `.columns`: 컬럼명 목록을 확인한다.
- `.dtypes`: 컬럼별 자료형을 확인한다.
- `pd.to_datetime()`: 문자열 날짜를 Pandas 날짜 타입으로 변환한다.
- `.sort_values()`: 특정 컬럼 기준으로 데이터를 정렬한다.
- `.is_monotonic_increasing`: 값이 오름차순인지 확인한다.
- `.isna()`: 각 값이 결측치인지 `True`/`False`로 확인한다.
- `.sum()`: 숫자 합계를 계산하며, Boolean에서는 `True` 개수를 세는 데 쓸 수 있다.
- `.loc[]`: 조건에 맞는 행과 열을 선택한다.
- `.any(axis=1)`: 각 행에 `True`가 하나라도 있는지 확인한다.
- `.duplicated()`: 중복 행 여부를 확인한다.
- `.describe()`: 숫자 컬럼의 기본 요약 통계를 보여준다.
- `.mean()`: 평균을 계산한다.
- `.min()`: 최솟값을 계산한다.
- `.max()`: 최댓값을 계산한다.
- `.plot(kind="line")`: 선 그래프를 그려 시간 흐름을 확인한다.
- `.plot(kind="scatter")`: 산점도를 그려 두 변수의 관계를 확인한다.

#### 예측 대상과 설명 변수 선택

- 예측 대상(target variable):
  - 역할: 모델이 맞히려고 하는 값이다.
  - 고르는 기준: "미래에 알고 싶은 값이 무엇인가?"에서 출발해야 한다.
  - 나쁜 기준: 결측치가 없거나 평균이 안정적이라는 이유만으로 고르는 것.
  - 좋은 기준: 경제적으로 예측할 의미가 있고, 미래 의사결정에 필요한 값인 것.
- 설명 변수(explanatory variable):
  - 역할: 예측 대상 값을 예측하는 데 사용할 정보다.
  - 고르는 기준: 예측 대상과 관련이 있을 가능성이 있고, 예측 시점에 실제로 알 수 있는 정보여야 한다.
  - 주의점: 그래프에서 관계가 있어 보여도 예측력이나 인과관계는 아직 확정할 수 없다.

수정된 실수:

- 처음에는 날짜 정렬 이유와 날짜 변환 이유가 섞였다.
- 이후 날짜 변환은 타입 문제, 날짜 정렬은 시간 순서 문제라고 구분했다.
- 처음에는 평균의 한계를 튀는 값 중심으로만 설명했지만, 이후 평균은 시간 순서를 보여주지 못한다는 점을 추가했다.
- 예측 대상 후보를 고를 때 데이터가 깨끗한지에 많이 의존했지만, 이후 예측 목적이 더 중요하다는 점을 기록했다.
